# VCPE Installation Guide ## [LXD documentation](https://documentation.ubuntu.com/lxd) ## [Diagram 1: end to end - environment](https://app.diagrams.net/#Uhttps%3A%2F%2Fraw.githubusercontent.com/robvogelaar/robvogelaar.github.io/main/drawio/containers.drawio "ctrl-click to open containers.drawio") ## [Diagram 2: vcpe layering](https://app.diagrams.net/#Uhttps%3A%2F%2Fraw.githubusercontent.com/robvogelaar/robvogelaar.github.io/main/drawio/mvx.drawio "ctrl-click to open mvx.drawio") ## Prerequisites The Virtual CPE Environment operates on an x86 Linux system using LXD containerization. VCPE runs a large number of Linux containers simultaneously, VCPE also creates specific networking configurations including network interfaces and bridges. While it can run on a shared server, deploying it on a dedicated physical host is recommended. ## OS Requirements Ubuntu 20.04: is recommended as it is known to work correctly for all cpe, bng, client, and services - container configurations. ## OS Prerequisites sudo without password prompt is recommended. ```bash sudo apt install jq yq graphviz python3 python3-pip sudo pip install msgpack ``` ## Install LXD from snap Note: LXD uninstall instructions for apt and snap can be found at the bottom of this page. ```bash sudo snap install lxd --channel=6.1 lxd init #select all defaults lxd version ``` Add your user to the LXD group: ```bash sudo usermod -a -G lxd $USER newgrp lxd ``` Run a container - internet connectivity test (test lxdbr0 NAT): ```bash lxc launch images:alpine/edge edge lxc exec edge -- ping google.com -c 3 lxc delete edge -f ``` ## Install meta-lxd meta-lxd contains a large number of scripts to manage the VCPE environment. This includes scripts required for container configuration and creation, these are found in meta-lxd/gen. meta-lxd also includes scripts required for running various vcpe tests, and scripts required for obtaining and processing vcpe data, including a large number of debug and visualization tooling scripts known as probes. These (probes) scripts are in meta-lxd/probes/scripts. meta-lxd/doc contains additional documentation. meta-lxd/recipes is part of the vcpe container build layers (yocto) meta-layers. Clone the repo: ```bash cd mkdir -p git cd git git clone ssh://git@bitbucket.upc.biz:7999/~rvogelaar/meta-lxd.git ``` Add meta-lxd to system PATH: ```bash vi ~/.bashrc export PATH="$HOME/git/meta-lxd/gen:$PATH" export PATH="$HOME/git/meta-lxd/probes/scripts:$PATH" source ~/.bashrc ``` # VCPE Nomenclature ## acs container The acs container (debian 7 based) contains the Axiros cwmp stack. cpe containers will register and establish the tr069 protocol with the acs. Auto testing (soap) acs scripts can be found in meta-lxd. ## usp container The usp container (debian 12 based) contains the Oktopus usp controller stack. cpe containers run usp agent and will register and establish the usp over mqtt protocol with Oktopus. Oktopus services run as docker containers. ## webpa container The webpa container (centos-stream 9 based) contains the webpa server xmidt stack. cpe containers run webpa client and will register and establish the web-pa protocol with web-pa server. ## bng containers The bng containers (devuan chimaera based) provide configuration and connectivity services to the cpe containers (similar to cmts / back-office). This includes: router.cfg tftp, dhcpv4/v6, radvd, dns, ntp, iperf, http, etc. There are multiple bng containers providing a specific real world market (customer index) configuration. For instance bng-9 is configured for multi-vlan with a specific ip-pool. Each bng provides a wan (internet) and a cm (cmts) connection to the cpe containers. bng containers provide internet acces to cpe containers using NAT. ## mvx (cpe) containers The mvx or cpe containers run the full ofw (rdk-b) router gateway stack, including oem and soc layers, with a virtual hal. These containers are created from the rootfs obtained from a rdk-b build server. When lxd starts a container, it runs the /sbin/init in the container. The cpe containers follow a strict naming scheme. This helps identify what is running in the container: MV-REL-BNG-EXTRA MV : mv1 mv2plus mv3 : ie. one of the common LG virtualized platforms REL : r21 : ie. the release number. Builds for r22 release have not been prepared BNG : 7 9 20 : this indicates to which bng (customer index) this container is configured and connected EXTRA : empty or 001..099, use the extra when creating another instance of a container cpe containers are created from a mv1, mv2plus, or mv3 image. They are then provided with a unique profile, this profile contains device configurations such as which network devices are required. cpe containers are also provided with a unique storage volume which holds the persistent nvram data. Since the name of the nvram storage volume that is associated with the container is configured in the container's profile, each container requires a unique profile. ## (lan) client containers clients are 'alpine' based, and run busybox - udhcpc. ## Bridges lxdbr1 bridge. This bridge connects the acs container to the bng containers, and provides host gateway (internet access) to the bng containers using NAT. cpe containers connect to bng containers using a common wan bridge (supporting single/multi/no - vlan) and a common cm bridge. lan clients connect to cpe containers lan ports using lan-p1..lan-p4 bridges. The cpe containers lan ports and the lan-p1.lan-p4 bridges are vlan configured allowing a large number of client connections to cpe containers lan ports. wlan interfaces connect to wlan0 and wlan1 bridges. ## Install CPE containers Run the mv.sh script to install mv1 container to work with bng-7 (no vlan): ```bash mv.sh https://vcpe-images.vcpe.dev/mv1-r21.rootfs 7 ``` Run the mv.sh script to install mv2plus container to work with bng-7 (no vlan): ```bash mv.sh https://vcpe-images.vcpe.dev/mv2plus-r21.rootfs 7 ``` Run the mv.sh script to install a second mv2plus container to work with bng-7 (no vlan), this container can later be configured to work in bridge mode: ```bash mv.sh https://vcpe-images.vcpe.dev/mv2plus-r21.rootfs 7 001 ``` Run the mv.sh script to install mv3 container to work with bng-9 (multi vlan): ```bash mv.sh https://vcpe-images.vcpe.dev/mv3-r21.rootfs 9 ``` Run the mv.sh script to install mv3 container to work with bng-9 (multi vlan), this container can later be configured to work in bridge mode: ```bash mv.sh https://vcpe-images.vcpe.dev/mv3-r21.rootfs 9 001 ``` Run the mv.sh script to install a second mv3 container to work with bng-20 (single vlan): ```bash mv.sh https://vcpe-images.vcpe.dev/mv3-r21.rootfs 20 ``` Run the mv.sh script to install a second mv3 container to work with bng-20 (single vlan), this container can later be configured to work in bridge mode: ```bash mv.sh https://vcpe-images.vcpe.dev/mv3-r21.rootfs 20 001 ``` ## Configure the CPE containers On each container run a factory default and set customer id on the containers to match with the bng. Do this by shell into the container and running the appropriate command syscfg / dmcli etc. ```bash lxc exec mv1-r21-7 exec bash root@mv1-r21-7:~$ hint run the appropriate command: e.e root@mv1-r21-7:~$ syscfg set Customer_Index 7 && syscfg commit root@mv1-r21-7:~$ reboot ``` Note: the cpe container bash history will be saved in /nvram/, to write the history file exit and re-enter the container, the history will now be available upon a container reboot. ## Install (lan) client containers Run the vcpe script with the 'create client' option. This will automatically create clients for all cpe containers. Each cpe container will receive a client on each of its lan ports ```bash vcpe create client ``` ## Container management: list all containers ```bash lxc list lxc image list lxc profile list lxc storage volume list ``` enter a cpe container's console ```bash lxc console mv2plus-r21-7 lxc console mv3-r21-9 bash ``` enter a shell on a cpe container ```bash lxc exec mv2plus-r21-7 bash lxc exec mv3-r21-9 bash lxc exec bng-7 bash lxc exec acs bash ``` run a command in a container ```bash lxc exec mv2plus-r21-7 bash -- dmcli eRT getv Device. ``` pull a file from a container ```bash lxc file pull mv3-r21-9/rdklogs/logs/WANManager.log.txt . ``` push a file to a container ```bash lxc file push debug.ini mv3-r21-9/etc/debug.ini ``` stop start restart (reboot) a container Note: stopping (restarting) a container is a graceful procedure and takes a few seconds to finish. If a container is in a state where it cannot shutdown cleanly often due to dhcp release, then the container can be forced stopped or restarted using the -f option. In this case ctrl-c twice to break the pending action and rerun the action with -f. ```bash lxc stop mv3-r21-9 lxc start mv3-r21-9 lxc restart mv3-r21-9 ``` force delete all lan clients ```bash lxm.sh client delete -f ``` # Probes ## build [cloc mv1](https://www.vcpe.dev/lxd/build/cloc-mv1-lxd-r21-oe30-1115.html "ctrl-click to open") [size mv3](https://www.vcpe.dev/lxd/build/size-mv3-lxd-r21-oe31-1115.html "ctrl-click to open") [meta mv2plus](https://www.vcpe.dev/lxd/build/meta-mv2plus-lxd-r21-oe31-1115.html "ctrl-click to open") [deps mv3](https://www.vcpe.dev/lxd/build/deps-mv3-lxd-r21-oe31-1115.html "ctrl-click to open") ## run [combined rdk logs](https://www.vcpe.dev/lxd/run/1120/mv3-r21-9/rdklogs/combined_logs.txt.0.html "ctrl-click to open") [datamodel](https://www.vcpe.dev/lxd/run/1120/mv3-r21-9/datamodel-mv3-r21-9.html "ctrl-click to open") [sysevent map](https://www.vcpe.dev/lxd/run/1120/mv3-r21-9/sysevent-mv3-r21-9.log.map.html "ctrl-click to open") [rbus](https://www.vcpe.dev/lxd/run/1120/mv3-r21-9/rbus-mv3-r21-9.html "ctrl-click to open") [syscfg](https://www.vcpe.dev/lxd/run/1120/mv3-r21-9/syscfg-mv3-r21-9.html "ctrl-click to open") [forkstat](https://www.vcpe.dev/lxd/run/1115/mv3/1115/forkstat-mv3-r21-9-eventviewer.html "ctrl-click to open") [rssfree](https://www.vcpe.dev/rssfree/rssfree-mv3-r16-0614.html "ctrl-click to open") ## getrdklogs ```bash getrdklogs.sh mv3-r21-9 ``` ## getlogs ```bash getlogs.sh mv3-r21-9 ``` ## forkstat ```bash do-forkstat.sh mv3-r21-9 ``` ## reboot ```bash do-reboot.sh ``` ## reload ```bash do-reload.sh 1124 ``` ## acs tests ## usp tests ## webpa tests # LXD UI [Reference](https://documentation.ubuntu.com/lxd/en/stable-5.21/tutorial/ui/) Manage containers using lxd web interface. ```bash lxc config set core.https_address :8443 ``` start a browser on port 8443, follow instructions. # Prometheus and Grafana see docs/metrics ## After a host reboot All containers will come up in the stopped state, they all require a lxc start.
